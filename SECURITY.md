# Security Policy

MyGameBuilder Local is in active development. Until stable releases are established, security support applies to the current `main` branch.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public GitHub issues, pull requests, comments, or community spaces.

Use GitHub private vulnerability reporting if it is enabled for the repository:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Submit the report through GitHub's private advisory form.

If private vulnerability reporting is not available, contact the maintainers privately before sharing details publicly.

## What to Include

Please include:

- a clear description of the issue;
- steps to reproduce it;
- affected branch, commit, release, or local setup, if known;
- expected and actual behavior;
- screenshots, logs, proof-of-concept code, or requests/responses, if useful;
- any suggested fix or mitigation.

Do not include sensitive personal data unless it is necessary to explain the issue.

## Public Issues

Use normal GitHub issues for ordinary bugs, feature requests, setup problems, documentation problems, and compatibility differences that do not expose users, data, infrastructure, or credentials to harm.
